Hateful Meme Detection

The model was pretrained on the ImageNet dataset to leverage its visual feature extraction capabilities, followed by retraining and fine-tuning on the Hateful Memes dataset to specialize in identifying subtle multimodal patterns of hate speech. By integrating textual and visual data, the model effectively interprets context-dependent cues that are critical for meme analysis.

To evaluate the modelâ€™s generalizability, a performance analysis was conducted on the MS COCO dataset, a benchmark multimodal dataset featuring diverse image-text pairs. This evaluation provided insights into the modelâ€™s strengths and limitations, enabling iterative modifications to the transformer architecture. These refinements optimized its performance, enhancing its ability to detect nuanced and context-sensitive hateful content in memes.

Real Estate Investment Guide

In this project, I developed a data-driven Real Estate Investment Guide, designed to empower investors by consolidating and analyzing property metrics across the U.S. This tool presents an interactive dashboard that delivers a suite of customizable metrics tailored for assessing real estate investments. Key investment metrics include property valuation trends, average and median rental yields, cash-on-cash return, cap rate, appreciation potential, vacancy rates, and local economic indicators. Additionally, the tool incorporates Airbnb data to estimate potential short-term rental income, enabling investors to compare long-term and short-term profitability.

To enhance usability, I implemented interactive filters, allowing users to view insights by state, city, or zip code level, as well as by property type and size. The interface dynamically adjusts to user-selected parameters, displaying data visualizations that provide a clear, comparative view of selected areas or property types. Users can also input their investment criteria, such as budget or target return rate, and receive real-time updates on matching properties, helping them quickly identify high-potential investments.

Technologies Used:

- Data Sources: Zillow API, World Population Review API, Airbnb dataset

- Python (Pandas, NumPy) for data cleaning and processing.

- PostgreSQL for structured data storage and management.

- Tableau for dynamic visualizations and interactivity

Multi Threaded HTTP Server

This project is a multi-threaded HTTP server implemented in C, designed to handle multiple concurrent client requests. It features basic request parsing, response formatting, and file handling, with support for concurrent client connections via POSIX threads and read-write locks to ensure safe file access. A thread pool processes incoming requests, where each thread retrieves client connections from a synchronized queue. Using a reader-writer lock mechanism, the server safely manages concurrent access to files, allowing multiple clients to read the same resource while maintaining exclusive access for writes.

The server parses HTTP requests, extracting methods, URIs, headers, and, when needed, validating formats with regular expressions. Depending on the request's result, the server generates responses with appropriate status codes (e.g., 200 OK, 403 Forbidden, 404 Not Found), and formats response headers. Error responses are generated for unsupported methods, bad formats, permission issues, and missing files. A Makefile is included to automate the compilation process, with targets for building, cleaning, and testing, streamlining development and deployment.

Technologies Used:

- C programming language for low-level network and file handling.

- POSIX Threads (pthreads) for multi-threading support and concurrency.

- Regular Expressions for parsing HTTP request headers.

- File I/O and Memory Management for efficient request processing and response generation.

- Read-Write Locks and Mutexes for thread-safe file operations.
